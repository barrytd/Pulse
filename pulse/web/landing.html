Get started

Three commands. From zero to live findings.

Pulse runs anywhere Python runs. Drop the dashboard on a $5/mo VPS, install agents on the hosts you care about, walk away.

Download the agent

Grab the packaged Windows bundle (37 MB, one-folder, no installer required).

pulse-agent · Windows x64

Enroll on the host

Mint an enrollment token in Settings → Agents, then trade it for a long-lived bearer:

pulse-agent enroll <url> <token>

Start the loop

The agent heartbeats every 60s and ships findings every 30 minutes by default.

pulse-agent run

Prefer source? Clone the repo and run pip install -e .. Server runs on Linux / macOS / Windows; agent is Windows-only.

🚧 Direct download isn't built on this preview server — grab the source from GitHub and run python scripts/build_agent.py --clean to produce pulse-agent.exe locally.

FAQ

Questions teams ask before deploying.

Is Pulse a SIEM, an EDR, or something else?
Pulse sits between the two. It's a Windows-event-log-focused detection platform: think SIEM-style rule engine + EDR-style fleet visibility, scoped to the Windows event log surface. If you already have Splunk + CrowdStrike and a six-figure budget, Pulse won't replace them. If you're running on event logs alone, Pulse covers the detection + triage + reporting flow without a per-event invoice.
Why Windows-only on the agent side?
The detection rules read .evtx files via wevtutil — that's a Windows-only API. The dashboard / server runs on Linux, macOS, or Windows; agents are Windows-only by design. macOS / Linux endpoint coverage is on the roadmap.
Can I write my own detection rules?
Yes — that's the point. Detections live in pulse/core/detections.py as plain Python functions. Add a function, tag it with a MITRE technique + severity, restart the server. The new rule shows up in the rule library and fires on the live monitor immediately. No DSL, no external rule compiler, no waiting for a vendor's content release.
How does the agent talk to the dashboard?
HTTPS only, three endpoints: /api/agent/exchange (one-time enrollment), /api/agent/heartbeat (every 60s), /api/agent/findings (every 30min by default). Two-token auth: a single-use enrollment token mints a long-lived bearer; both stored sha-256-at-rest. Pause or revoke from the dashboard and the agent stops at the next heartbeat.
Is there telemetry?
Zero. Pulse never phones home. The auto-update channel is opt-in and points at whatever URL you configure (defaults to GitHub Releases on the public repo). Self-hosted deploys can override every URL via env vars.
What's the multi-tenant story?
Every owned row (scans, agents, notifications) carries an organization_id. Self-signup mints a fresh org; admin-side user creation joins the admin's org. Two customers running on the same hosted Pulse instance never see each other's data, but every member of the same org shares scan / agent / finding visibility — the right collaboration boundary for a SOC team.
Can I run Pulse air-gapped?
Yes. The server has no required external dependencies. AbuseIPDB enrichment is opt-in (drop in your own API key); auto-update can point at an internal URL. Email / Slack / Discord alerting is optional. Everything ships in a single Python package + SQLite (or Postgres if you prefer) database.
